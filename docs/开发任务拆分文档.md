# yt-dlp Windows 桌面下载管理器 V1 开发任务拆分文档

> 文档版本：V1.0  
> 项目类型：Windows 桌面程序  
> 技术栈：Electron + React + Vite + Node.js + yt-dlp + FFmpeg  
> 目标平台：Windows 10 / Windows 11  
> 文档目标：将 V1 版本拆分为可执行、可验收、可交给 Codex / DeepSeek / 开发人员逐步完成的开发任务。

---

## 1. V1 版本定位

V1 是整个项目的基础版本，目标不是一次性实现 yt-dlp 的所有高级能力，而是先完成一个稳定、可运行、可打包、可下载的 Windows 桌面程序。

V1 的核心目标是：

```text
用户打开桌面程序
输入视频链接
解析视频信息
选择下载视频或音频
选择保存目录
点击下载
看到下载进度
下载完成后打开文件夹
```

V1 必须优先保证核心链路稳定：

```text
Electron 启动
React 页面显示
IPC 通信正常
yt-dlp 可调用
FFmpeg 可调用
视频可下载
音频可提取
进度可显示
任务可管理
配置可保存
历史可记录
Windows 可打包
```

## 3. V1 开发阶段总览

建议将 V1 拆成 8 个阶段：

```text
阶段 1：项目初始化与基础框架
阶段 2：Electron 主进程与安全 IPC
阶段 3：yt-dlp / FFmpeg 二进制资源管理
阶段 4：视频解析功能
阶段 5：单任务下载功能
阶段 6：任务中心与进度显示
阶段 7：配置、历史与批量下载
阶段 8：错误处理、测试、打包发布
```

推荐开发顺序：

```text
框架 → IPC → 路径检测 → 解析 → 下载 → 进度 → 任务 → 配置 → 历史 → 批量 → 打包
```

## 17. 总结

V1 的核心不是把 yt-dlp 所有功能都做出来，而是完成一个稳定的最小可用桌面下载器。

V1 最关键的 5 条链路是：

```text
1. Electron + React 能启动
2. React 通过 IPC 调用主进程
3. 主进程能调用 yt-dlp
4. yt-dlp 下载进度能回传 UI
5. 下载完成后能保存历史并打开文件夹
```

只要 V1 稳定，V2 才能放心扩展播放列表、字幕、封面、Cookie、代理、archive、命令预览等专业功能。

V1 开发时必须坚持：

1. 少做功能。
2. 保证链路。
3. 保证安全。
4. 保证路径兼容。
5. 保证打包可用。
6. 保证错误可理解。
7. 为 V2 留出清晰扩展点。
