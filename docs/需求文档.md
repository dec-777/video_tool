# yt-dlp Windows 桌面下载管理器需求文档

> 文档版本：V1.0  
> 项目类型：Windows 桌面程序  
> 技术方向：Electron + React + yt-dlp + FFmpeg  
> 目标平台：Windows 10 / Windows 11  
> 核心定位：将 yt-dlp 的命令行能力封装为可视化桌面下载管理器

---

## 1. 项目背景

yt-dlp 是一个功能强大的命令行音视频下载工具，支持 YouTube、Bilibili、TikTok、Twitch、Vimeo、Twitter/X 等大量网站，也支持播放列表、频道、合集、课程、番剧分 P、字幕、封面、元数据、Cookie 登录、代理、下载记录等高级能力。

但 yt-dlp 本身主要面向命令行用户，普通用户在使用时存在以下问题：

1. 命令参数复杂，学习成本高。
2. Windows 用户不熟悉命令行操作。
3. 下载进度、任务管理、历史记录不直观。
4. 批量下载、播放列表下载、Cookie、代理、字幕、封面等功能配置繁琐。
5. 音视频合并依赖 FFmpeg，用户容易配置错误。
6. 下载失败时错误信息难以理解。
7. 文件命名、保存路径、去重下载等功能不够可视化。

因此，本项目计划开发一个 Windows 桌面程序，为 yt-dlp 提供专业、易用、可视化的前端 UI。

---

## 2. 项目目标

本项目目标是开发一个基于 Electron + React 的 Windows 桌面下载管理器，通过可视化界面调用本地 yt-dlp.exe 和 FFmpeg.exe，实现视频、音频、字幕、封面、元数据、Cookie、代理、文件管理、下载历史等功能。

### 2.1 核心目标

1. 用户无需使用命令行即可使用 yt-dlp。
2. 支持单个 URL、多 URL、播放列表、频道、合集、课程、番剧分 P 等下载场景。
3. 支持视频、音频、字幕、封面、元数据等多类型资源下载。
4. 支持清晰度、格式、编码、容器、音频格式选择。
5. 支持 Cookie 登录、浏览器 Cookie、代理下载。
6. 支持自定义保存目录、文件名模板、下载记录、跳过已下载。
7. 支持任务队列、进度显示、失败重试、取消任务、日志查看。
8. 支持 Windows 打包安装，内置 yt-dlp.exe、ffmpeg.exe、ffprobe.exe。

## 4. 用户角色

### 4.1 普通用户

普通用户希望通过简单界面输入视频链接，选择格式并下载到本地。

需求：

1. 操作简单。
2. 下载过程可视化。
3. 下载完成后能直接打开文件夹。
4. 下载失败时能看到清晰原因。

高级用户需要使用 yt-dlp 的高级能力。

需求：

1. 指定清晰度、编码、格式 ID。
2. 使用 Cookie 下载登录后内容。
3. 使用代理。
4. 下载字幕、封面、元数据。
5. 自定义文件名模板。
6. 使用 archive 跳过已下载。
7. 查看原始命令和日志。

### 4.3 开发者 / 维护者

开发者需要清晰的架构和可扩展模块。

需求：

1. 功能模块清晰。
2. UI 与本地后端解耦。
3. yt-dlp 参数生成逻辑独立。
4. 任务管理逻辑独立。
5. 后续可扩展站点模板、自动更新、批量管理等能力。

## 5. 技术架构

### 5.1 技术栈

| 层级 | 技术 |
|---|---|
| 桌面框架 | Electron |
| 前端框架 | React |
| 构建工具 | Vite |
| UI 状态管理 | Zustand / Redux Toolkit |
| 本地后端 | Electron Main Process + Node.js |
| 进程调用 | child_process.spawn |
| 下载核心 | yt-dlp.exe |
| 音视频处理 | FFmpeg / FFprobe |
| 配置存储 | JSON 文件，后续可升级 SQLite |
| 打包工具 | electron-builder |
| 目标平台 | Windows 10 / Windows 11 |

### 5.2 总体架构

```text
用户
 ↓
React 前端 UI
 ↓
preload.js 安全桥接
 ↓
Electron IPC
 ↓
Electron Main Process 本地后端
 ↓
commandBuilder.js 生成 yt-dlp 参数
 ↓
ytdlpService.js 调用 yt-dlp.exe
 ↓
FFmpeg 合并 / 转码 / 后处理
 ↓
本地文件系统
```

### 5.3 前后端职责划分

#### React 前端负责

1. 展示界面。
2. 接收用户输入。
3. 展示解析结果。
4. 展示下载任务。
5. 展示下载进度。
6. 展示历史记录。
7. 展示设置项。
8. 展示命令预览。
9. 通过 IPC 调用本地后端。

#### Electron Main Process 负责

1. 调用 yt-dlp.exe。
2. 调用 FFmpeg。
3. 生成命令参数。
4. 管理下载进程。
5. 管理下载队列。
6. 读取和保存配置。
7. 保存任务历史。
8. 打开文件或文件夹。
9. 处理 Windows 路径。
10. 返回错误信息和日志。

## 7.1 首页下载模块

### 7.1.1 功能说明

首页作为快速下载入口，用户可以输入单个视频链接，解析视频信息，选择下载模式，并开始下载。

### 7.1.2 UI 元素

1. URL 输入框。
2. 解析按钮。
3. 直接下载按钮。
4. 添加到队列按钮。
5. 视频封面展示。
6. 视频标题展示。
7. 作者 / 上传者展示。
8. 视频时长展示。
9. 站点名称展示。
10. 下载模式选择。
11. 清晰度选择。
12. 保存目录选择。
13. 命令预览区域。

### 7.1.3 功能逻辑

用户输入 URL 后点击“解析”，系统调用：

```bash
yt-dlp.exe -J "URL"
```

解析成功后，前端显示：

1. 标题。
2. 封面。
3. 作者。
4. 时长。
5. 站点。
6. 可用格式。
7. 字幕信息。
8. 是否为播放列表。

### 7.1.4 验收标准

1. 输入合法 URL 后可以成功解析基础信息。
2. 解析失败时显示明确错误信息。
3. 用户可以选择下载模式并开始下载。
4. 下载任务进入任务中心。
5. 命令预览能实时展示当前生成的 yt-dlp 命令。

### 7.4.2 下载模式

### 7.4.3 视频质量

1. 最佳画质。
2. 最低画质。
3. 指定分辨率。
4. 指定格式 ID。
5. 原始格式。

### 7.4.4 容器格式

1. mp4。
2. webm。
3. mkv。
4. 原始容器。

### 7.4.5 音频格式

1. mp3。
2. m4a。
3. opus。
4. wav。


```bash
--merge-output-format mp4
```

提取 MP3：

```bash
-x --audio-format mp3
```

6. 下载后文件格式符合用户选择。

## 7.5 音视频分离与合并模块

### 7.5.1 功能说明

部分网站的视频流和音频流是分开的，本系统需要通过 FFmpeg 自动合并。

### 7.5.2 功能要求

1. 内置 ffmpeg.exe。
2. 内置 ffprobe.exe。
3. 下载命令自动添加 FFmpeg 路径。
4. 支持合并为 mp4、webm、mkv。
5. 支持合并后删除中间文件。
6. 支持用户选择保留中间文件。

### 7.5.3 对应参数

```bash
--ffmpeg-location ./bin
--merge-output-format mp4
--keep-video
```

### 7.5.4 验收标准

1. 下载 1080p 及以上视频时能正常合并音频。
2. 合并后文件有画面也有声音。
3. 用户选择保留中间文件时，中间音视频流不被删除。
4. FFmpeg 缺失时能给出明确提示。

## 7.10 文件管理模块

### 7.10.1 功能说明

支持自定义保存目录、自定义文件名模板、自动分文件夹、跳过已下载、保留或删除中间文件。

### 7.10.2 保存目录

用户可以选择默认下载目录。

示例：

```text
D:/Downloads
```

### 7.10.3 文件名模板

默认模板：

```text
%(title)s.%(ext)s
```

作者 + 标题：

```text
%(uploader)s - %(title)s.%(ext)s
```

按站点 / 作者分文件夹：

```text
%(extractor)s/%(uploader)s/%(title)s.%(ext)s
```

播放列表模板：

```text
%(playlist_title)s/%(playlist_index)s - %(title)s.%(ext)s
```

### 7.10.4 跳过已下载

通过 archive 文件记录已经下载过的视频。

对应参数：

```bash
--download-archive archive.txt
```

### 7.10.5 中间文件处理

合并后默认删除中间文件，用户可选择保留。

对应参数：

```bash
--keep-video
```

### 7.10.6 验收标准

1. 可以选择保存目录。
2. 可以设置文件名模板。
3. 文件名模板能正确传入 yt-dlp。
4. 开启 archive 后重复视频会跳过。
5. 可以选择是否保留中间文件。

## 12. 非功能需求

### 12.1 性能要求

1. 软件启动时间尽量控制在 5 秒内。
2. 单任务下载过程 UI 不得卡顿。
3. 支持至少 100 条历史记录展示。
4. 支持至少 100 个批量 URL 导入。
5. 默认并发数为 1，最大并发数建议不超过 5。

### 12.2 稳定性要求

1. yt-dlp 进程异常退出时，主程序不能崩溃。
2. 下载失败时任务状态必须正确更新。
3. 配置文件损坏时应自动恢复默认配置。
4. FFmpeg 缺失时应提示用户。
5. 软件关闭时应妥善处理未完成任务。

### 12.3 安全要求

1. 不允许前端执行任意系统命令。
2. 不暴露 Node.js API 给 React 页面。
3. Cookie 信息不得上传。
4. Cookie 信息不得写入普通日志。
5. 用户输入的 URL、路径、代理地址必须校验。
6. 所有命令参数必须以数组方式传给 spawn。
7. 不使用 exec 拼接命令字符串。

### 12.4 易用性要求

1. 常用功能放在首页。
2. 高级功能放在设置页或高级面板。
3. 每个高级选项需要有简短说明。
4. 下载失败时显示人类可理解的错误原因。
5. 下载完成后提供“打开文件夹”按钮。
6. 命令预览帮助高级用户理解参数。

## 17. 风险与解决方案

### 17.1 网站规则变化

风险：视频网站规则变化导致下载失败。

解决方案：

1. 提供 yt-dlp 更新功能。
2. 显示清晰错误信息。
3. 提供日志导出。
4. 提醒用户更新下载核心。

### 17.2 FFmpeg 缺失

风险：音视频无法合并。

解决方案：

1. 内置 FFmpeg。
2. 启动时检测 FFmpeg。
3. 缺失时提示用户。
4. 设置页允许手动指定路径。

### 17.3 Cookie 安全

风险：Cookie 是登录凭证，泄露会影响账号安全。

解决方案：

1. Cookie 只本地使用。
2. 不上传 Cookie。
3. 不在日志显示 Cookie 内容。
4. UI 明确提示风险。

### 17.4 杀毒软件误报

风险：打包 yt-dlp.exe 和 ffmpeg.exe 可能被误报。

解决方案：

1. 使用官方来源二进制文件。
2. 软件说明中解释组件用途。
3. 后续考虑代码签名证书。
4. 避免可疑脚本行为。

### 17.5 Windows 路径问题

风险：中文路径、空格路径导致命令失败。

解决方案：

1. 使用 spawn 参数数组。
2. 不使用 exec 拼接命令。
3. 所有路径使用 path.join。
4. 测试中文路径和空格路径。

## 20. 结论

本项目的本质是：

```text
用 Electron + React 做 Windows 桌面 UI，
用 Electron Main Process 做本地后端，
用 yt-dlp.exe 做下载核心，
用 FFmpeg 做音视频合并和转码，
用本地 JSON / SQLite 管理配置、任务和历史。
```

项目成功的关键不在于重新实现下载能力，而在于：

1. 把 yt-dlp 参数可视化。
2. 把复杂命令转换成简单 UI。
3. 把下载任务管理做好。
4. 把错误提示做好。
5. 把文件管理做好。
6. 把 Cookie、代理、字幕、封面等高级功能做成可选配置。

建议按照 V1、V2、V3 分阶段开发，先保证核心下载链路稳定，再逐步扩展高级功能。
